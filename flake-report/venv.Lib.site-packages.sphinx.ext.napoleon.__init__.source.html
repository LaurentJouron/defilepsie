349 """Process the docstring for a given python object.
350  
351 Called when autodoc has read and processed a docstring. `lines` is a list
352 of docstring lines that `_process_docstring` modifies in place to change
353 what Sphinx outputs.
354  
355 The following settings in conf.py control what styles of docstrings will
356 be parsed:
357  
358 * ``napoleon_google_docstring`` -- parse Google style docstrings
359 * ``napoleon_numpy_docstring`` -- parse NumPy style docstrings
360  
361 Parameters
362 ----------
363 app : sphinx.application.Sphinx
364 Application object representing the Sphinx process.
365 what : str
366 A string specifying the type of the object to which the docstring
367 belongs. Valid values: "module", "class", "exception", "function",
368 "method", "attribute".
369 name : str
370 The fully qualified name of the object.
371 obj : module, class, exception, function, method, or attribute
372 The object to which the docstring belongs.
373 options : sphinx.ext.autodoc.Options
374 The options given to the directive: an object with attributes
375 inherited_members, undoc_members, show_inheritance and no_index that
376 are True if the flag option of same name was given to the auto
377 directive.
378 lines : list of str
379 The lines of the docstring, see above.
380  
381 .. note:: `lines` is modified *in place*
382  
383 """
384 result_lines = lines
385 docstring: GoogleDocstring
386 if app.config.napoleon_numpy_docstring:
387 docstring = NumpyDocstring(result_lines, app.config, app, what, name,
388 obj, options)
389 result_lines = docstring.lines()
390 if app.config.napoleon_google_docstring:
391 docstring = GoogleDocstring(result_lines, app.config, app, what, name,
392 obj, options)
393 result_lines = docstring.lines()
