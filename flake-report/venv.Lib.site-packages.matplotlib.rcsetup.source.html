1 """
2 The rcsetup module contains the validation code for customization using
3 Matplotlib's rc settings.
4  
5 Each rc setting is assigned a function used to validate any attempted changes
6 to that setting. The validation functions are defined in the rcsetup module,
7 and are used to construct the rcParams global object which stores the settings
8 and is referenced throughout Matplotlib.
9  
10 The default values of the rc settings are set in the default matplotlibrc file.
11 Any additions or deletions to the parameter set listed here should also be
12 propagated to the :file:`lib/matplotlib/mpl-data/matplotlibrc` in Matplotlib's
13 root source directory.

477 def _is_iterable_not_string_like(x):
478 # Explicitly exclude bytes/bytearrays so that they are not

488 offset = 0
489 onoff = ls

662 def cycler(*args, **kwargs):
663 """
664 Create a `~cycler.Cycler` object much like :func:`cycler.cycler`,
665 but includes input validation.
666  
667 Call signatures::
668  
669 cycler(cycler)
670 cycler(label=values[, label2=values2[, ...]])
671 cycler(label, values)
672  
673 Form 1 copies a given `~cycler.Cycler` object.
674  
675 Form 2 creates a `~cycler.Cycler` which cycles over one or more
676 properties simultaneously. If multiple properties are given, their
677 value lists must have the same length.
678  
679 Form 3 creates a `~cycler.Cycler` for a single property. This form
680 exists for compatibility with the original cycler. Its use is
681 discouraged in favor of the kwarg form, i.e. ``cycler(label=values)``.
682  
683 Parameters
684 ----------
685 cycler : Cycler
686 Copy constructor for Cycler.
687  
688 label : str
689 The property key. Must be a valid `.Artist` property.
690 For example, 'color' or 'linestyle'. Aliases are allowed,
691 such as 'c' for 'color' and 'lw' for 'linewidth'.
692  
693 values : iterable
694 Finite-length iterable of the property values. These values
695 are validated and will raise a ValueError if invalid.
696  
697 Returns
698 -------
699 Cycler
700 A new :class:`~cycler.Cycler` for the given properties.
701  
702 Examples
703 --------
704 Creating a cycler for a single property:
705  
706 >>> c = cycler(color=['red', 'green', 'blue'])
707  
708 Creating a cycler for simultaneously cycling over multiple properties
709 (e.g. red circle, green plus, blue cross):
710  
711 >>> c = cycler(color=['red', 'green', 'blue'],
712 ... marker=['o', '+', 'x'])
713  
714 """
715 if args and kwargs:
716 raise TypeError("cycler() can only accept positional OR keyword "
717 "arguments -- not both.")
718 elif not args and not kwargs:
719 raise TypeError("cycler() must have positional OR keyword arguments")
720  
721 if len(args) == 1:
722 if not isinstance(args[0], Cycler):
723 raise TypeError("If only one positional argument given, it must "
724 "be a Cycler instance.")
725 return validate_cycler(args[0])
726 elif len(args) == 2:

804 # arbitrary code without sanitation.

807 # I locked it down by only having the 'cycler()' function available.
